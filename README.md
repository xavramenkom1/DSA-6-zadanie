# Задание 6 – Бинарные решающие диаграммы (BDD)

Создайте программу, в которой можно построить структуру данных BDD (Binary Decision Diagram) для представления булевых функций.

## Требуемые функции

Реализуйте:

- `BDD *BDD_create (string bfunkcia, string poradie);`
- `BDD *BDD_create_with_best_order (string bfunkcia);`
- `char BDD_use (BDD *bdd, string vstupy);`

Можно добавлять вспомогательные функции, но **нельзя использовать готовые библиотеки для BDD**.

---

## 1. `BDD_create`

Функция строит **редуцированную BDD** для заданной булевой функции.

- `bfunkcia` — булево выражение (формат выбираете сами, например DNF-строка)
- `poradie` — порядок переменных

Возвращает: указатель на BDD.

### Требования к структуре BDD:
- количество переменных
- размер (число узлов)
- указатель на корень

Также нужна структура узла (вершины).

### Важно:
- Обязательна **редукция BDD**
- Лучше делать редукцию **во время построения** (иначе штраф −3 балла)
- Работать нужно с выражением, а не с векторами (иначе −3 балла)

---

## 2. `BDD_create_with_best_order`

Ищет **лучший порядок переменных**.

Идея:
- несколько раз вызвать `BDD_create` с разными порядками
- выбрать BDD с минимальным числом узлов

Минимум: проверить хотя бы **N различных порядков**, где N — число переменных.

Возможные подходы:
- циклические сдвиги
- все перестановки (N!)
- случайные порядки

Возвращает: лучший BDD.

---

## 3. `BDD_use`

Использует BDD для вычисления результата.

- `bdd` — BDD
- `vstupy` — строка значений (например `"1010"`)

Работа:
- пройти от корня до листа по значениям переменных

Возвращает:
- `'1'` или `'0'`
- при ошибке: отрицательное значение (например `-1`)

---

## Тестирование

Обязательно:

- Генерировать **случайные булевые функции**
- Проверять:
  - все комбинации входов
  - сравнение с реальным вычислением выражения

Требования:
- минимум 100 функций на одно число переменных
- поддержка минимум 13 переменных

### Метрики:
- % редукции:
  - `(удалённые узлы / полный BDD)`
- улучшение от лучшего порядка
