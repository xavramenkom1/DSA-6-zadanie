# Задание 6 – Бинарные диаграммы решений (BDD)

## Описание задания

Создайте программу, в которой можно будет создать структуру данных BDD (Binary Decision Diagram — Бинарная диаграмма решений) с акцентом на использование для представления булевых функций.

---

## Реализуемые функции

```c
BDD *BDD_create(string bfunkcia, string poradie);
BDD *BDD_create_with_best_order(string bfunkcia);
char BDD_use(BDD *bdd, string vstupy);
```

Вы также можете реализовать вспомогательные функции, однако **запрещается** использовать готовые библиотечные функции для работы с бинарными диаграммами решений.

---

## Описание функций

### `BDD_create`

Служит для построения **редуцированной** бинарной диаграммы решений, которая представляет заданную булеву функцию.

- **Аргумент `bfunkcia`** — булева функция в виде выражения (формат и тип данных на ваш выбор, например DNF-строка).
- **Аргумент `poradie`** — порядок переменных, определяющий, в каком порядке используются переменные при построении BDD (формат на ваш выбор: связный список, массив переменных, пронумерованных от 0 до N-1 и т.д.).
- **Возвращаемое значение** — указатель на построенную (и редуцированную) BDD, представленную собственной структурой `BDD`.

#### Структура `BDD` должна содержать минимум:
- количество переменных,
- размер BDD (количество узлов),
- указатель на корень (первый узол) BDD.

Также необходима собственная структура для представления одного узла BDD (вершины дерева).

#### Редукция:
- Редукцию можно выполнить **после** построения полного BDD или **в процессе** его построения.
- ⚠️ Если редукция выполняется только после полного построения — **штраф 3 балла** (более низкая эффективность).
- ⚠️ Функция должна работать напрямую с **выражениями**, а не с векторами — иначе **штраф 3 балла**.

---

### `BDD_create_with_best_order`

Служит для нахождения **наилучшего порядка переменных** для заданной булевой функции.

- Метод: многократный вызов `BDD_create` с разными порядками переменных.
- Примеры стратегий:
  - Циклические сдвиги (для 5 переменных: `01234`, `12340`, `23401`, `34012`, `40123`),
  - Все возможные перестановки — N! (для 5 переменных: 5! = 120),
  - X случайных уникальных порядков.
- **Требование:** проверить не менее **N уникальных порядков**, где N — количество переменных.
- **Возвращаемое значение** — указатель на BDD с **наименьшим количеством узлов** из всех проверенных порядков.

---

### `BDD_use`

Служит для применения BDD к конкретной комбинации значений входных переменных и получения результата булевой функции.

- **Аргумент `bdd`** — указатель на BDD.
- **Аргумент `vstupy`** — строка/массив символов, представляющая конкретную комбинацию значений переменных.  
  Пример: для переменных A, B, C, D, где A=1, B=0, C=1, D=0 → `"1010"`.
- **Возвращаемое значение:**
  - `'1'` или `'0'` — результат булевой функции,
  - отрицательное значение (например, `-1`) — в случае ошибки (некорректный ввод).

---

## Тестирование

- Необходимо **случайным образом генерировать** булевы функции и строить по ним BDD с помощью `BDD_create` и `BDD_create_with_best_order`.
- Корректность проверяется **итеративным вызовом** `BDD_use` для всех возможных комбинаций входных переменных с последующим сравнением с ожидаемым результатом (вычисленным путём подстановки значений в выражение).
- Тестировать на **разном количестве переменных** (минимум **13 переменных**).
- Количество различных булевых функций / BDD-диаграмм для одного числа переменных — минимум **100**.
- Оценивать:
  - **Процент редукции BDD** = (удалённые узлы) / (узлы полной диаграммы).
  - **Дополнительный процент редукции** от перебора порядков = сравнение размера BDD из `BDD_create` и `BDD_create_with_best_order`.

## Пример простого теста

```c
#include <string.h>

int main() {
    BDD* bdd;
    bdd = BDD_create("AB+C", "012");

    if (BDD_use(bdd, "000") != '0')
        printf("Ошибка: для A=0, B=0, C=0 результат должен быть 0.\n");
    if (BDD_use(bdd, "001") != '1')
        printf("Ошибка: для A=0, B=0, C=1 результат должен быть 1.\n");
    if (BDD_use(bdd, "010") != '0')
        printf("Ошибка: для A=0, B=1, C=0 результат должен быть 0.\n");
    if (BDD_use(bdd, "011") != '1')
        printf("Ошибка: для A=0, B=1, C=1 результат должен быть 1.\n");
    if (BDD_use(bdd, "100") != '0')
        printf("Ошибка: для A=1, B=0, C=0 результат должен быть 0.\n");
    if (BDD_use(bdd, "101") != '1')
        printf("Ошибка: для A=1, B=0, C=1 результат должен быть 1.\n");
    if (BDD_use(bdd, "110") != '1')
        printf("Ошибка: для A=1, B=1, C=0 результат должен быть 1.\n");
    if (BDD_use(bdd, "111") != '1')
        printf("Ошибка: для A=1, B=1, C=1 результат должен быть 1.\n");

    return 0;
}
```
